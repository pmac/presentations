Demo Servers

  • We have as many as we need
  • Some are pre-defined
    • e.g. www-demo1.allizom.org
  • Others are for each developer
    • e.g. demo-agibson.us-west.moz.works
  • Still others are named however we want
Any developer with access to push to our Github repository can push to or create a demo.
### Dev Server Dev refers to `www-dev.allizom.org` 1. Is updated with every push to `master` branch * e.g. a pull-request is merged 2. All Feature Switches are `on` 3. The [in-development l10n files](https://github.com/mozilla-l10n/www.mozilla.org) are used
### Staging Servers * Refers to `www.allizom.org` * Updated at the same time as production * Meant as a test of deployment and should always match Production as closely as possible
### Production Servers There are several stages to a production deploy: 1. Test all of the code 2. Deploy to Staging 3. Run integration tests against Staging 4. Deploy to Production 5. Run integration tests against Production 6. Do steps 2 - 5 again for other server cluster
